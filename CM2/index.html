Le concept d'une library

  • Une library n'est rien d'autre que du code contenu dans "d'autres" fichiers
  • Je simplifie à l'excès mais ce n'est pas si loin de la vérité
  • L'idée centrale, votre code doit être structuré de manière logique, pas tout dans un seul fichier !

Comment importer depuis un autre fichier (ou d'une library)

  • import name_of_library
  • Dans ce cas, chaque fois que nous voudrons utiliser les ressources de la library importée, nous devrons la faire précéder de name_of_library.
  • from name_of_library import name_of_what_to import
  • Dans ce cas, nous pouvons utiliser name_of_what_to directement
  • from name_of_library import *

    Les choix

    Jeu de Nim

    Problème :
    On a besoin de pouvoir vérifier que le nombre entré par l'utilisateur est correct..
    Mais aussi que le nombre restant de batonnets est $0$ ou non...
    • Vous pouvez définir des conditions pour exécuter uniquement des parties du code si les conditions sont vraies
    • Les instruction "if" permet de coder des alternatives sur la base d’expressions booléennes
    • REMARQUE : L'indentation est utilisée pour séparer le code à l'intérieur d'un choix

    L’expression booléenne peut être construite à l’aide d’opérateurs de comparaison

    • x == y est vraie si x est égal à y
    • x != y est vraie si x est différent de y
    • x > y est vraie si x est plus grand que y
    • x < y est vraie si x est plus petit que y
    • x >= y est vraie si x est plus grand que, ou égal à y
    • x <= y est vraie si x est plus petit que, ou égal à y
    • Il y a aussi l'opérateur "is", mais nous verrons cela plus tard

    Regardons un exemple

    
>>> x = eval("Choisissez un entier positif: ")
>>> if x < 0:
>>>   print("C'est pas positif !")

    Et si vous voulez vérifier plusieurs conditions alternatives ?

    • Les booléens peuvent être combinés à l'aide d'opérateurs logiques
    • or pour des conditions alternatives
    • and pour des conditions qui doivent être à la fois vraies 
    
>>> x = eval("Choisissez un entier positif mais pas plus de 10: ")
>>> if x < 0 or x > 10:
>>>   print("C'est pas positif et moins de 10!")
						
    
>>> if x >= 0 or x <= 10:
>>>   print("C'est un bon choix")
    • La deuxième condition est-elle nécessaire ?
    • C'est toujours vrai si la première ne l'est pas...

    On peut utiliser elif ou else pour combiner plusieurs solutions alternatives

    
>>> x = eval("Choisissez un entier positif: ")
>>> if x < 0:
>>>   print("C'est pas positif !")
>>> elif x > 10:
>>>   print("C'est trop grand !")
>>> else:
>>>   print("C'est un bon choix")

    REMARQUE : Une seule de ces branches s'exécutera !

    Exceptions et erreurs

    • Les erreurs, ça arrive assez souvent dans la vie !
    • Aussi en programmation (en général même très souvent 😊 )
    • Les erreurs peuvent être dues à des distractions (l'indentation par exemple...)
    • Ou à cause d'erreurs dans les instructions doonées à l'interpréteur
    • Ou de fausses hypothèses

    L'exemple le plus simple

    
>>> print(y)
Traceback (most recent call last):
  File "< stdin >", line 1, in < module >
NameError: name 'y' is not defined

    Peut-être que vous n'avez pas considéré les valeurs possibles d'une variable

    
>>> 1/0
Traceback (most recent call last):
	File "< stdin >", line 1, in < module >
ZeroDivisionError: division by zero

    Ou simplement vous avez un problème avec votre algorithme

    
>>> a = []
>>> a[1]
Traceback (most recent call last):
	File "< stdin >", line 1, in < module >
IndexError: list index out of range
    • Lorsqu'une erreur se produit, Python arrête toute exécution de code en cours
    • Mais aussi, il fournit des outils pour permettre à un programmeur de récupérer d'une erreur sans arrêter son programme
    • En utilisant la syntaxe try ... except pour capturer une exception
    • Les exceptions sont des parties de code qui ne sont exécutées que lorsqu'une erreur se produit

    Vous spécifiez avec try la partie du code que vous souhaitez couvrir en capturant une exception, et except la partie du code qui sera exécutée si l'erreur se produit

    
>>> x = eval("Choisissez un entier positif qui ce n'est pas 0 : ")
>>> try:
>>>   result = 1/x
>>> except ZeroDivisionError:
>>>   print('Erreur: a ne doit pas être égal à 0')
    • Que se passe-t-il si vous êtes dans un try mais un autre type d'erreur se produit ?

    Vous pouvez utiliser une condition générale

    
>>> x = eval("Choisissez un entier positif qui ce n'est pas 0 : ")
>>> try:
>>>   result = y/x
>>> except Exception:
>>>   print("Je l'ai capturée :)")

    REMARQUE : Les exceptions générales ne sont normalement pas utilisées car elles ne vous permettent pas de savoir quel problème s'est produit !

    Regroupement d’instructions

    Jeu de Nim

    Problème :
    Pour chaque joueur on doit faire la même chose. Doit on ré-écrire deux fois exactement le même code ?
    • Parfois, une partie du code doit être réutilisée plusieurs fois
    • Par exemple, pour tester différentes valeurs dans l'exemple précédent
    • En Python, le regroupement s’effectue à l’aide des fonctions auxquelles il est possible de fournir des paramètres
    • Une fonction est un bloc de code qui est "appelé" ou "invoqué" en écrivant son nom et en spécifiant les paramètres dont il a besoin

    Une fonction est définie à l'aide du mot clé def suivi du nom de la fonction et d'une liste de paramètres

    
>>> def check_value(x):
>>>   if x < 0:
>>>     print("C'est pas positif !")
>>>   elif x > 10:
>>>     print("C'est trop grand !")
>>>   else:
>>>     print("C'est un bon choix")

    REMARQUE : N'oubliez pas l'indentation !

    On peut utiliser la fonction assez simplement dans l’interpréteur de commande

    
>>> check_value(1)
>>> check_value(-1)
>>> check_value(100)
>>> check_value("ciao")

    REMARQUE : N'oubliez pas l'indentation !

    • Les fonctions peuvent également produire un résultat à utiliser après leur appel
    • Ce résultat est "renvoyé" en utilisant le mot-clé return suivi de la valeur à retourner
    • Une valeur peut être spécifiée directement ou nous pouvons utiliser le contenu d'une variable

    L’utilisation de return permet de préciser la valeur retournée par la fonction

    
>>> def check_value(x):
>>>   if x < 0:
>>>     print("C'est pas positif !")
>>>     return False
>>>   elif x > 10:
>>>     print("C'est trop grand !")
>>>     return False
>>>   else:
>>>     print("C'est un bon choix")
>>>     return True
>>>
>>> if check_value(10):
>>>   print("La valeur était ok")

    REMARQUE : N'oubliez pas l'indentation !

nombre_batonnets = nombre_batonnets - nombre_pris

if (nombre_pris <=0 or nombre_pris>min(3, nombre_batonnets)):
	print("Le nombre de bâtonnets à prendre n'est pas correct !")
	print('Choisir une valeur entre 1 et ', min(3, nombre_batonnets), '.')
	nombre_pris = int(input("Entrez un nombre de bâtonnets :"))

nombre_batonnets -= nombre_pris

if nombre_batonnets == 0:
	print("Félicitations !! Vous avez gagné !")

# ------------------------------------------------------------------------
# 7 - Fonction pour prendre le choix et vérifier les condiitons sans 
# prise en compte du numéro du joueur
# ------------------------------------------------------------------------
def choix_joueur(nombre_batonnets):
	nombre_pris = int(input("Entrez un nombre de bâtonnets : "))

	# Il faudrait une répétition mais ça on verra après !
	if (nombre_pris <=0 or nombre_pris>min(3, nombre_batonnets)):
		print("Le nombre de bâtonnets à prendre n'est pas correct !")
		print('Choisir une valeur entre 1 et ', min(3, nombre_batonnets), '.')
		nombre_pris = int(input("Entrez un nombre de bâtonnets :"))

	return nombre_pris

# ------------------------------------------------------------------------
# 8 - Fonction pour faire un tour de jeu avec prise en compte joueur
# ------------------------------------------------------------------------
def tour_jeu(no_joueur, nombre_batonnets):
	print("Joueur no.", no_joueur)
	nombre_pris = choix_joueur(nombre_batonnets)

	nombre_batonnets -= nombre_pris
	if nombre_batonnets == 0:
		print("Félicitations !! Vous avez gagné !")
		return 0
	else:
		print("Il reste ", nombre_batonnets, "batonnets.")
		return nombre_batonnets


nombre_batonnets = 12
nombre_batonnets = tour_jeu(1, nombre_batonnets)
nombre_batonnets = tour_jeu(2, nombre_batonnets)
